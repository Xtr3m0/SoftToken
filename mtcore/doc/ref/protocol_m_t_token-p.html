Detailed Description

This is the protocol implemented by the token objects.

On the Gemalto Ezio OATH Client API for iOS, a token object is the OTP token itself. It is used to generates the OTP.

Instances of this protocol must be loaded from the core using the method MTAppCore::loadTokenForAccount:.

5.0

BOOL MTToken-p::calculateNextEventOTP ( ) [virtual]

Calculates the next event OTP.

Returns:
YES for success or NO otherwise.
Warning:
Each call to this method increments the event counter, thus multiple calls to this method without a successfull authentication will make the token out of synchronization.
The behavior of this method is undefined if the event OTP is not available.

float MTToken-p::getCycleProgress ( ) [virtual]

This method calculates the fraction of the current time cycle.

The main purpose of this method the implementation of a gauge that keeps track of the current time cycle.

BOOL MTToken-p::updateTimeOTP ( ) [virtual]

Updates the time OTP calculated by MTToken::calculateTimeOTP:.

This method calculates the if, and only if, the previous OTP is no longer valid. This makes this method suitable for interactive interfaces.

Returns:
YES for success or NO otherwise.
Note:
The method MTToken::calculateTimeOTP: must be called at least once before this method can be invoked.
