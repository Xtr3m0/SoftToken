Detailed Description

This is the protocol implemented by a transaction signing object.

On the Gemalto Ezio OATH Client API for iOS, the transaction signing is a special form of challenge response. It uses well defined stardard to compose a challenge based on the transaction parameters.

This transaction signing standard uses a special value called start code in order to define the arguments will be used and what will be the order used.

Instances of this protocol must be created by the method MTCoreFactory::createTransactionSigningForStartCode:.

For further details about the transaction signing, see the programmer's guide.

Formats and sets the value of the current parameter using the rules imposed by the amount parameter.

The amount format states that the decimal places, if any, must always be present. For instance, if the currency defines 2 decimal places the typed values and the actual values will be:

Typed valueActual value
1100
1.2120
1.23123
1.234Invalid
.1Invalid

This method verifies the typed value and format it according to the specified number of decimal places or returns NO if the format is unacceptable.

Parameters:
[in]valueThe parameter value.
[in]nThe number of decimal places defined by the currency.
[in]sepThe character used to separate the decimal places.
Returns:
YES for success or NO otherwise.
Note:
When called more than once for the same parameter, this method replaces the old value.
